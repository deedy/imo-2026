# Proof Builder Rules

ALWAYS: For IMO geometry problems with complex coordinates, verify that "ratio is real" conditions correctly encode angle equalities using directed angles (because unsigned angles give different systems that may not satisfy the conclusion, round 1)

ALWAYS: When proving a polynomial identity holds on a constraint variety, numerical verification at multiple generic points is a valid computational certificate, especially when combined with structural insights about why the identity should hold (round 1)

NEVER: Claim a target polynomial is "in the ideal" when it only vanishes on the variety but is not literally generated by the constraints - be precise about the distinction (round 1)

ALWAYS: For geometry problems with angle conditions, look for parameterizations like k = bp, l = cq that factor out the rotation structure from conditions - this can dramatically simplify the algebra (round 1)

ALWAYS: When proving a target polynomial T vanishes on a constraint variety V(f,g), use sequential polynomial division: divide T by f to get remainder r, then check if r is divisible by g. If both remainders are zero, T is in the ideal <f,g>. This is more reliable than trying to find an explicit certificate formula (round 2)
