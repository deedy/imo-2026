# Proof Reviewer Memory

## Rules

ALWAYS: Verify claimed answer computationally for game theory problems (round 1)
ALWAYS: Check both lower bound AND upper bound separately for optimization problems (round 1)
ALWAYS: When a proof contains self-corrections or "let me reconsider" sections, verify the final conclusion is correct (round 1)
ALWAYS: When a proof claims a pigeonhole/dichotomy lemma, construct explicit counterexamples before accepting (round 2, Lemma 5 failure)
NEVER: Accept a saddle-point argument where one of the conditions relies on a false lemma (round 2)
ALWAYS: When reviewing a case-split induction, verify ALL cases are covered before accepting as solved (round 3)
ALWAYS: When a "create-duplicate" or "pair-cancellation" strategy is used, independently verify parity preservation (shift by 2 preserves odd/even) and that pairs actually occupy top positions (round 4)
