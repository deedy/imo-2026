# proof-reviewer role notes

ALWAYS: Verify the key algebraic identities independently using sympy or direct computation before approving (round 2)
ALWAYS: Check numerical examples given in proofs by hand/computation - a single numerical contradiction can validate an entire case (round 2)
ALWAYS: Trace through "Case 1" and "Case 2" separately even if they look similar - they often have subtly different logic (round 2)
NEVER: Accept surjectivity claims without proof when the argument depends on it - Range(f) may have gaps (round 2)
