# outline-reviewer rules

ALWAYS: Verify algebraic identities with sympy before accepting them (because hand-waving algebra in outlines can hide errors, round 2)
ALWAYS: Test orbit-growth arguments numerically with edge cases like a=0 (because the a=0 case in orbit-invariance required a different sub-argument, round 2)
ALWAYS: Flag when approaches share the same framing and only differ in technique - they fail together if the shared step is wrong (round 2)
NEVER: Accept a "gap" label without a stated mechanism - push back on load-bearing lemmas that say "prove X" without explaining why X is true (round 2)
