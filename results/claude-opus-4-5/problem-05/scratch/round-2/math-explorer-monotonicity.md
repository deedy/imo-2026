# IMO 2026 P5 — Monotonicity/Convexity Lens Report

## Problem
Find all f: ℝ₊ → ℝ₊ such that
  √((x²+f(y)²)/2) ≥ (f(x)+y)/2 ≥ √(x·f(y))   for all x, y > 0.

## Distinct openings

**Opening A (Main route — g-constant via orbit comparison):**
Set g = f − id. The two inequalities expand in g-form as:
- L = (x−y)² + 2g(y)f(y) − g(x)(g(x)+2(x+y)) ≥ 0    [left ineq in g-form]
- R = (x−y)² + g(x)² + 2g(x)(x+y) − 4xg(y) ≥ 0       [right ineq in g-form]

Key algebraic identities (verified numerically):
- L + R = 2(x−f(y))²
- L − R = 2(g(y)−g(x))(f(x)+x+f(y)+y)

Since both L ≥ 0 and R ≥ 0, we get min(L,R) = (L+R−|L−R|)/2 ≥ 0, which gives:

  **(x − f(y))² ≥ |g(x)−g(y)| · (f(x)+x+f(y)+y)   for all x,y > 0.   (**)**

Now suppose g(x₀) = a and g(y₀) = b with 0 ≤ a < b. By orbit invariance, f^n(x₀) = x₀+na and g(x₀+na)=a; similarly for y₀. Apply (**) with x = x₀+n_t·a and y = y₀+m_t·b where n_t = round(bt), m_t = round(at):
- LHS = (x₀−y₀ + n_t·a − (m_t+1)b)² ≤ C² (bounded, since n_t·a − m_t·b = O(a+b))
- RHS ≥ (b−a)·(2x₀ + 2n_t·a + …) ~ (b−a)·4abt → ∞

For large t: RHS > C², contradiction. So a = b; g is constant.

**Opening B (Squeeze from sup/inf of bounds):**
For fixed x, the right inequality forces f(x) ≥ 2√(xf(y)) − y for all y (lower envelope), while the left forces f(x) ≤ √(2(x²+f(y)²)) − y (upper envelope). Both bounds are tight simultaneously at the unique y* = f⁻¹(x) where f(y*) = x. This self-pinning means f is the fixed point of a Legendre-type transform: f(x) = sup_y[2√(x·f(y)) − y].

**Opening C (Hölder-2 regularity from (**) near Range(f)):**
Fix z ∈ Range(f), say f(y₀) = z. Set x = z+t in (**): |g(z+t)−g(z)| ≤ t²/(4z+O(t)). So g is Hölder-2 (derivative zero) at every z ∈ Range(f). Since f(y) ∈ Range(f) for all y and g(f(y)) = g(y) by orbit invariance, this forces g constant on Range(f), hence everywhere.

**Opening D (Non-orbit comparison via the a=0 fixed-point case):**
Even if a = 0 (fixed point x₀), (**) with x near x₁ > x₀ and y near x₀ forces |x₁−x₀|² ≥ b·(4x₀+b). So the 'gap' between g=0 and g=b regions must be at least 2√(bx₀), which numerical evidence shows is violated by piecewise functions. Confirmed: f_piece(x) = x for x≤1, x+1 for x>1 violates both L and (**) at (x,y) = (1.5, 0.5).

## Candidate technique(s)

- **Functional identity from equality-case substitution**: x = f(y) forces both inequalities to equality, giving f(f(y)) = 2f(y)−y.
- **Orbit invariance + algebraic manipulation of both inequalities simultaneously**: The key is computing L+R and L−R explicitly, then extracting the combined bound (**).
- **Contradiction via orbit growth**: Two orbits with different g-values eventually put the RHS of (**) to infinity while the LHS stays bounded — this is a clean algebraic argument that requires no regularity assumption on f.

## Cheap-kill candidates

- **Descent**: g(x) < 0 somewhere → f^n(y₀) = y₀ + n·g(y₀) → −∞, violates f: ℝ₊ → ℝ₊. So g ≥ 0.
- **f(x) = kx forced to k=1**: From f(f(y)) = 2f(y)−y with f = k·id: k² = 2k−1, so k=1. Quickly kills purely multiplicative candidates.
- **QM−AM failure for super-linear f**: f(x) = x+√x fails LEFT inequality numerically (tested). The LEFT inequality gives an upper-bound: if f has super-linear growth anywhere, QM(x,f(y)) can be smaller than AM(f(x),y) for small y.

## Dead ends (do not retry)

- **f(x)/x = constant approach**: Setting u=f(x)/x and treating the right inequality as a quadratic in x gives spurious results because u varies with x. Argument breaks down.
- **f(x) = x is the only solution**: Wrong. f(x) = x+c works for ALL c ≥ 0.
- **f(x) = 2x**: Numerically fails LEFT inequality (e.g., x=2, y=1).
- **f(x) = x+√x**: Numerically fails LEFT inequality for small y.
- **Piecewise f = id on (0,x₀] and f=id+c on (x₀,∞)**: Fails at (x,y)=(1.5,0.5); violation confirmed both via direct check and via (**).

## Small-case / intuition notes

**Answer (conjectured, very strong evidence):** f(x) = x + c for any constant c ≥ 0.

Verified numerically for c ∈ {0, 0.5, 1, 2, 5} on grids including small/large values.

Equality in both inequalities holds simultaneously if and only if x = f(y) (i.e., on the curve {(x,y): f(y)=x}).

**Key algebraic identities (verified):**
- L + R = 2(x−f(y))²   [this is always non-negative, consistent]
- L − R = 2(g(y)−g(x))(f(x)+x+f(y)+y)   [the sign encodes the ordering of g-values]
- (**): (x−f(y))² ≥ |g(x)−g(y)|·(f(x)+x+f(y)+y)   [the combined constraint]

**Orbit formula:** f^n(y) = y + n·g(y) if g is constant on the orbit (which is forced by orbit invariance once g(y) is determined).

**The clean route to f = x+c:**
1. Substitute x=f(y): get f(f(y))=2f(y)−y.
2. Set g=f−id. Get g(f(y))=g(y) (orbit-invariant).
3. g≥0 (descent argument).
4. Compute L+R and L−R to get (**).
5. If g takes two distinct values a<b, the orbit growth argument (n_t=round(bt), m_t=round(at)) gives LHS bounded but RHS→∞ in (**). Contradiction.
6. So g is constant c≥0; f(x)=x+c.
7. Verify f(x)=x+c satisfies both: left iff (x−(y+c))²≥0 ✓; right iff (x−y−c)²≥0 ✓.

**Remaining gap for a rigorous proof:** The orbit-growth argument in Step 5 needs n_t and m_t to be non-negative integers (which they are for t≥1). The key estimate |n_t·a − m_t·b| ≤ (a+b)/2 must be verified carefully from the definition n_t=round(bt), m_t=round(at). This is straightforward.
