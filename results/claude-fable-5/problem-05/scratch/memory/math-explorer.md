ALWAYS: for FE/inequality "determine all f" problems, symbolically verify (sympy factor/simplify) any candidate family across ALL relevant parameters before assuming the answer is just f=identity — for imo-2026-05, f(x)=x+c for every c>=0 turned out to satisfy both sandwich inequalities exactly (perfect-square identity), so the "obvious" unique-identity answer was wrong; round 1 caught this only via direct sympy factoring, not by guessing.
ALWAYS: when a functional inequality is squeezed between two bounds (sqrt(...)>=...>=sqrt(...)), try substituting one variable equal to f(other variable) — it collapses one outer expression to a bare f-value and often yields an EXACT equation (not just an inequality) by combining the two halves of the sandwich; this cracked f(f(y))=2f(y)-y in imo-2026-05.
NEVER: trust that setting x=y in a two-variable "sandwich" functional inequality gives useful info — check first whether it reduces to a tautological AM-GM/QM-AM instance (true for ANY f), which wastes a proof branch.
