# proof-builder rules

ALWAYS: replace outline limit-arguments with explicit concrete bounds (pick n satisfying a stated inequality, bound O(1) terms by constants) — reviewers flag "→ −∞" style steps as gaps (round 1).
ALWAYS: state squaring/root-taking legality (positivity of every term) once at the top when a problem's hypothesis is a chain of radicals — it is a mandatory rigor item for QM/AM/GM-shaped inequalities (round 1).
ALWAYS: when proposing a lemma file under results/<id>/lemmas/, mark it "PROPOSED (uncertified)" with a pointer to where it is proved — only the reviewer certifies (round 1).
