# proof-outliner — per-role rules

ALWAYS: for gcd/lcm blackboard processes, keep invariance arguments per-prime but termination
arguments global — one move couples all primes, per-prime counters are not monovariants
(round 1).
NEVER: assume the two outputs of the (gcd, lcm/gcd) move are coprime — false, 4,6 -> 2,6
(round 1).
ALWAYS: when the field's favorite is an exact-invariant proof, make the fallback a different
ENGINE (e.g. confluence/Newman), not a different invariant — different walls (round 1).
